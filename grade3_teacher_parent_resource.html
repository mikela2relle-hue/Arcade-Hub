Detailed Curriculum Expectations and Design Philosophy.

🧠 Curriculum Structure & Design Philosophy

Welcome to the detailed curriculum guide! This resource provides a comprehensive overview of the Ontario Grade 3 Mathematics curriculum, focusing on the strands covered by the Brain Boost Arcade activities (currently B1, B2, and C1).

**Why the Interactive Learning Doesn't Follow Strict Order:** You might notice that the **interactive learning activities** sometimes mix expectations (e.g., placing B2.3 Estimation early) or group related skills together (e.g., B1.2 Comparison and B1.3 Rounding). This is intentional! The activities prioritize a **logical learning flow** for a Grade 3 student over strict numerical sequence. Foundational skills are often grouped, and related concepts are presented together to reinforce understanding and simulate real classroom assessment practices. This tree view below, however, presents the curriculum in its official, structured order for your reference.

🔢 B. Number

▼

B1: Number Sense

▼

B1.1 Read, represent, compose, and decompose whole numbers up to 1000...

Focus: Understanding numbers to 1000 in different forms (words, numerals, expanded: $600 + 30 + 8$) and using tools like base ten blocks or number lines.

Examples:

  • Words: "six hundred thirty-eight"
  • Standard: 638
  • Expanded: $600 + 30 + 8$
  • Base Ten: 3 flats, 2 rods, 7 units for 327
  • Number Line: Representing numbers as points or distances.

Key Concepts:

  • Place value determines a digit's value (e.g., the 4 in 4107 is 4 thousands).
  • Numbers can be broken down (decomposed) and built up (composed) in many ways (e.g., $512 = 250 + 250 + 10 + 2$).
  • Tools help visualize number representation.
  • Numbers describe quantities ("how much?").

Note:

  • Conservation principle: Quantity remains unchanged when decomposed/recomposed.
  • Flexibility in decomposing supports mental math (e.g., $587 = 58$ tens $+ 7$ ones).
  • Number lines (closed, open) are key tools.
  • Connect concrete models (base ten) with written forms.

Sample Tasks:

  • Show different ways to reach 638 on an open number line.
  • "Which Number Am I?" clues using place value or composition (e.g., "My number has 16 tens and 16 ones").
  • Represent 647 with base ten blocks, then regroup (e.g., 5 flats, 14 rods, 7 units).

B1.2 Compare and order whole numbers up to 1000...

Focus: Using strategies like benchmark numbers (e.g., comparing to 500) and place value to compare ($<, >, =$) and order numbers.

Examples:

  • Using Benchmarks: 512 is closer to 500 than 465.
  • Using Place Value: Compare 535 and 542 by looking at the tens place (3 vs 4).
  • Ordering: Ascending (65, 128, 456) or Descending (832, 456, 128).

Key Concepts:

  • Numbers are compared by magnitude ("how muchness").
  • Compare digits in the greatest place value where they differ.
  • Ascending = least to greatest; Descending = greatest to least.

Note:

  • Place value is key to comparing any numbers.
  • Metre stick (mm scale) can be a 0-1000 number line.

Sample Tasks:

  • Number line clothesline: Pin number cards in order using benchmarks.
  • "Close, Far, or In Between": Compare three numbers (e.g., 237, 384, 429).
  • Use a metre stick (mm) to locate and compare numbers.

B1.3 Round whole numbers to the nearest ten or hundred...

Focus: Understanding rounding for estimation, using the midpoint rule (5 or more rounds up).

Examples:

  • Context: Estimating sums, measuring (56mm ≈ 60mm or 6cm), paying with cash.

Key Concepts:

  • Rounding makes numbers simpler for estimation.
  • Rounding to tens is more precise than rounding to hundreds.
  • Midpoint Rule: Look at the digit to the right of the place you're rounding to. 5 or more rounds up, less than 5 rounds down. (e.g., 237 rounds to 240 (nearest ten) or 200 (nearest hundred)).

Note:

  • Precision needed determines how much to round.

Sample Tasks:

  • Number line clothesline with benchmarks (multiples of 10 or 100) and halfway marks. Pin numbers to the nearest ten/hundred.
  • Real-life rounding scenarios (e.g., how many $10 bills for a $32 purchase?).

B1.4 Count to 1000, including by 50s, 100s, and 200s...

Focus: Developing fluency in counting sequences, especially across hundreds, and skip counting by larger units.

Examples:

  • Skip counting by 200s on a number line.

Key Concepts:

  • Counting has a purpose (how many, how long, compare).
  • Can start counting from any number.
  • Skip counting unit = number of objects in a group.
  • The 0-9 sequence repeats within decades and hundreds.

Note:

  • Skip counting builds foundation for multiplication/division.
  • Counting across hundreds reinforces place value patterns.

Sample Tasks:

  • Count by ones across hundreds (e.g., 196 to 230).
  • Financial scenario: How to make $680 with $20, $50, $100 bills?
  • Create a class 0-1000 chart; use it for skip counting; find hidden numbers.

B1.5 Use place value when describing and representing multi-digit numbers...

Focus: Deepening understanding of how a digit's position (place) determines its value, using tools like base ten blocks.

Examples:

  • Using materials (blocks, bead strings) to show 1, 10, 100, 1000.
  • 10x10 grid showing relationship between 1, 10, 100.
  • Base ten blocks showing the value of '5' in 500, 50, and 5.

Key Concepts:

  • Place determines value (5 in 511 means 500).
  • Order matters (385 vs 853).
  • Zero is a placeholder (57 vs 507).
  • Expanded notation shows place value ($987 = 900 + 80 + 7$).

Note:

  • "Times ten" pattern between places ($50 = 10 \times 5$, $500 = 10 \times 50$).
  • Base ten materials are crucial.
  • Place value is foundational for calculations.
  • Hundreds-Tens-Ones pattern repeats in larger periods (thousands, millions).

Sample Tasks:

  • Compare/contrast numbers like 905, 509, 950, 590 (digit vs value).
  • Calculator task: Change '6' in 467 to '0' by subtracting the correct value (60, not 6).
  • "Number Wipe Out" calculator game (guessing digits and their values).
  • Model 777 on grid paper; discuss "times ten" relationship between areas.

B1.6 Use drawings for fair-share problems (up to 20 items among 2, 3, 4, 5, 6, 8, 10 sharers)...

Focus: Solving division problems where the result might be a whole number, fraction, or mixed number, using visual representations.

Examples:

  • Drawings showing 16 ribbons shared among 2, 3, or 6 friends.

Key Concepts:

  • Fair share = equal share.
  • Results can be whole numbers (4 shared by 2 = 2 each), fractions (4 shared by 8 = 1/2 each), or mixed numbers (4 shared by 3 = 1 and 1/3 each).
  • Comparing shares: More sharers = smaller share (if total is same); More items = bigger share (if sharers are same).

Note:

  • Clarify "fair" means "equal" in math context.
  • Connects naturally to division and fractions.
  • Introduce fractional notation ($\frac{1}{3}$) alongside words ("one third").

Sample Tasks:

  • Solve word problems: 16 waffles shared by 5 friends? 15 samosas by 10 friends? 18 strings by 10 students? Model with drawings/strips.
  • Compare scenarios: 13 bars among 8 friends vs. 13 bars among 5 friends? Who gets more?
  • Equalizing shares: If 2 friends share 4 plums, how many plums do 6 friends need to share to get the same amount each? Model and discuss ratio relationship.

B1.7 Represent and solve fair-share problems focusing on equivalent fractions...

Focus: Using fair-share contexts to discover and represent equivalent fractions (halves/fourths/eighths; thirds/sixths; fifths/tenths).

Examples:

  • Drawing: 5 friends share 1 flatbread (each gets $\frac{1}{5}$). If each friend splits their piece in 2, they now have $\frac{2}{10}$, showing $\frac{1}{5} = \frac{2}{10}$.

Key Concepts:

  • Equivalent fractions represent the same quantity ($\frac{1}{2} = \frac{2}{4} = \frac{4}{8}$).
  • Understanding unit fractions (fifths, tenths, etc.).

Note:

  • Fair share provides context for equivalence.
  • Patterns exist between equivalent fractions (e.g., numerator and denominator are both doubled).
  • Constant multiplicative relationship between equivalent ratios/fractions.

Sample Tasks:

  • Pose sharing problems leading to equivalent answers: ___ people share ___ items. Describe the share using two different fractions (e.g., 2 shared by 8 = $\frac{2}{8}$ or $\frac{1}{4}$). Discuss patterns.
  • Revisit B1.6, Task 3 (equalizing shares) and explicitly model the equivalent fractions involved.

B2: Operations

▼

B2.1 Use properties of operations and relationships (mult/div)...

Focus: Applying properties like commutative ($a \times b = b \times a$) and distributive ($4 \times 6 = (4 \times 5) + (4 \times 1)$) and the inverse relationship between multiplication and division to solve problems and check answers.

Examples:

  • Commutative (Mult): Show $4 \times 5 = 5 \times 4$ with arrays.
  • Distributive: Show $4 \times 6$ on ten frames as $(4 \times 5) + (4 \times 1)$.
  • Inverse: $10 \div 2 = ?$ can mean "how much in each group?" or "how many groups of 2?".

Key Concepts:

  • Multiplication/Division involve equal groups.
  • They are inverse operations ($16 \div 2 = ?$ is same as $? \times 2 = 16$).

Note:

  • Connect concrete models to properties (students don't need to name properties but should use them).
  • Supports many other Number expectations.

Sample Tasks:

  • Use tools to show $4 \times 8 = 8 \times 4$.
  • Model $6 \times 4$ on a rekenrek using $(5 \times 4) + (1 \times 4)$.
  • Connect multiplication/division representations ($3 \times 5 = 15$ and $15 \div 3 = 5$).
  • Memory game matching multiplication fact, division fact, and representation.

B2.2 Recall and demonstrate multiplication facts of 2, 5, and 10, and related division facts.

Focus: Building automaticity with key multiplication facts (2s, 5s, 10s) and understanding their connection to division through skip counting and visual models.

Examples:

  • Connecting skip counting by 2s to $\times 2$ facts.
  • Using arrays or equal groups to show facts like $4 \times 10 = 40$.
  • Using multiplication to solve division: $35 \div 5 = ?$ think $5 \times ? = 35$.
  • Strategy: $4 \times 5$ is half of $4 \times 10$.

Key Concepts:

  • Multiplication/Division are inverse operations.

Note:

  • Concrete models build understanding.
  • Automatic recall frees up working memory for complex problems.
  • Doubling, halving, skip counting are foundational strategies.
  • Multiplication involves a "double count" (groups and total).

Sample Tasks:

  • Model $\times 10$ and $\times 5$ facts on rekenrek/ten frames, notice $\times 5$ is half of $\times 10$.
  • Find patterns for 2s, 5s, 10s facts on a multiplication grid. Connect $\times 4$ to "double double".
  • Memory game matching fact (e.g., $2 \times 3$), representation (array), and answer (6).
  • Number strings connecting related facts ($\times 2$, $\times 5$, $\times 10$).

B2.3 Use mental math strategies, including estimation, to add and subtract up to 1000...

Focus: Developing flexible strategies for calculating addition/subtraction in one's head, sometimes using jottings, and using estimation (rounding) to check reasonableness.

Examples (Strategies):

  • Addition: Counting on ($3 + 789$), Making 10 ($684+7 \rightarrow 680+4+6+1$), Compensation ($869+15 \rightarrow 870+15-1$).
  • Subtraction: Counting on ($982-977$), Counting back ($692-687$), Subtracting in parts ($216-27 \rightarrow 216-20-7$), Back down through 10 ($93-5 \rightarrow 93-3-2$), Compensation ($978-480 \rightarrow 980-480-2$), Using Addition ($452-137 \rightarrow 137+?=452$).

Key Concepts:

  • Mental math = calculation in head (may use jottings).
  • Tools like number lines can show strategies.
  • Estimation (rounding) checks reasonableness.

Note:

  • Strategies vary based on numbers involved.
  • Goal is flexibility and understanding, not just speed.
  • Builds on basic facts and number relationships.

Sample Tasks:

  • Number Talks: Share strategies for problems like $398+123$. Visualize with number lines.
  • Mental subtraction task: Solve $452-137$ mentally, share strategies (using addition, breaking apart, compensation).
  • Apply strategies to real-world estimation (cost of purchase, making change).

B2.4/B2.5 Demonstrate understanding of +/- algorithms & solve problems up to 1000...

Focus: Connecting concrete tools (like base ten blocks) to the standard paper-and-pencil algorithms for addition and subtraction, understanding the "regrouping" or "trading" process based on place value, and solving various word problem types.

Examples (Algorithms & Situations):

  • Connecting base ten blocks to standard addition algorithm (regrouping 11 tens as 1 hundred, 1 ten).
  • Connecting base ten blocks to standard subtraction algorithm (trading 1 rod for 10 ones, 1 flat for 10 rods - "borrowing").
  • Problem Types: Change (Join/Separate), Combine (Part/Part/Whole), Compare (Difference).

Key Concepts:

  • Standard algorithms use place value for compact calculation.
  • Addition/Subtraction situations involve changing, combining, or comparing quantities.
  • Models (drawings, part-whole diagrams) help visualize problems.

Note:

  • Algorithm = steps for a procedure. Standard algorithms vary culturally.
  • Emphasize place *value* (e.g., "20 + 80") not just face *value* ("2 + 8") when explaining algorithms.
  • Matching equations to problem structures reinforces meaning. Part-whole models show inverse relationship and build algebraic reasoning.

Sample Tasks:

  • Model computations (e.g., $479+269$, $452-269$) with base ten blocks and explicitly link each step (trading/regrouping) to the written algorithm.
  • Investigate alternative addition/subtraction algorithms.
  • Solve various word problems (Change, Combine, Compare) using models, drawings, and algorithms. Write matching number sentences.
  • Given a scenario (e.g., 387 items total, 136 are cans, how many boxes?), write multiple equivalent equations to represent it ($387 - ? = 136$, $136 + ? = 387$, $387 - 136 = ?$).

B2.6/B2.7 Represent and solve multiplication (up to 10x10) and division (up to 100÷10)...

Focus: Using tools (especially arrays and number lines) and drawings to model and solve multiplication and division problems, including those involving simple fractions (halves, thirds, fourths).

Examples:

  • Arrays: Show $3 \times 4$ as 3 rows of 4. Show $36 \div 4 = ?$ by arranging 36 counters with 4 in each column.
  • Number Lines: Show $6 \times 5$ as 6 hops of 5. Show $35 \div 5$ as backward hops of 5 from 35.
  • Fractions: $12$ hops of $\frac{1}{3}$ equals 4. Subtracting $\frac{1}{4}$ seven times from $1 \frac{3}{4}$.

Key Concepts:

  • Multiplication finds total (known groups & size).
  • Division finds group size (sharing) or number of groups (grouping).
  • Array model shows rows/columns structure.
  • Division types: Equal Sharing (known total & #groups, find size); Equal Grouping (known total & size, find #groups).
  • Repeated addition/subtraction are common strategies.

Note:

  • Arrays visualize skip counting, distributive property, inverse relationship.
  • Rekenrek, ten frames, relational rods are also useful tools.
  • Representations help solve problems by counting; connect to equations.

Sample Tasks:

  • Create arrays for scenarios (e.g., 9 rows of 6 corn plants). Count creatively (skip counting, chunking using distributive property). See both $6 \times 9$ and $9 \times 6$.
  • Model division scenarios with counters (e.g., 48 band members in rows of 6. How many rows?). Use skip counting/known facts.
  • Solve equal-group word problems (product unknown, group size unknown, number of groups unknown). Model, use repeated +/-. Write mult/div sentences.
  • Solve fraction multiplication/division problems with drawings/number lines: $5$ weeks using $\frac{3}{4}$ can each week? How many plants needing $\frac{1}{3}$ can be watered with 5 cans? Connect counting unit fractions to repeated +/- and mult/div equations.

B2.8 Represent connection between numerator and repeated addition of unit fraction...

Focus: Understanding that the numerator counts the number of unit fractions being added together, using visual tools and standard notation.

Examples:

  • Fraction strips showing $\frac{1}{5} + \frac{1}{5} + \frac{1}{5} + \frac{1}{5} = \frac{4}{5}$.

Key Concepts:

  • Denominator: Bottom number, size of each part (unit fraction). More parts = smaller unit fraction.
  • Numerator: Top number, counts the equal parts (unit fractions).
  • A fraction like $\frac{2}{5}$ represents $\frac{1}{5} + \frac{1}{5}$.

Sample Tasks:

  • Model $\frac{4}{5}$ with a paper strip. Explain what '4' and '5' mean. Relate model to $\frac{1}{5}+\frac{1}{5}+\frac{1}{5}+\frac{1}{5} = \frac{4}{5}$.
  • Compose fractions like four-fifths using unit fractions and represent with tools/notation.

B2.9 Use ratios of 1:2, 1:5, and 1:10 to scale up numbers and solve problems.

Focus: Understanding and applying simple multiplicative ratios (doubling, 5 times, 10 times) in contexts like scaling and comparisons.

Examples:

  • Ratio 1 blue square to 2 green circles.
  • Pictograph key: 1 smiley = 5 students.
  • Double number line showing 1 matches 10.

Key Concepts:

  • Ratios describe multiplicative relationships (e.g., twice as many, 5 times as many).

Note:

  • Ratio notation $a:b$.
  • Scaling up = multiplying by a factor. Double number lines help visualize scaling.

Sample Tasks:

  • Solve ratio problems: Elise has 8 marbles. Esteban has 5 times as many. How many does Esteban have?
  • Draw a diagram showing 10 times as many milkweeds as coneflowers.
  • Redraw a pictograph with a 5:1 key using a 1:1 key.
  • Use double/triple number lines to show scaling relationships (e.g., cm to mm is 1:10). Compare 40 on scales of 1:10, 1:5, 1:2.

C1: Patterns and Relationships

▼

C1.1 Identify and describe repeating elements and operations...

Focus: Recognizing patterns in various contexts (nature, crafts, numbers) based on repeating elements (shapes, colors) or repeating operations (+, -, ×, ÷).

Examples:

  • Crafts: Beadwork, knitting sequences.
  • Nature: Honeycombs, phases of the Moon.
  • Numeric: Adding 25 cents every sixth day.

Key Concepts:

  • Pattern Core: Shortest string of repeating elements.
  • Repeating Operation: The mathematical action repeated (+5, ×2, etc.).

Note:

  • Patterns connect to culture and history.
  • Focus on what stays the same and what changes.
  • Operations (+0, -0, ×1, ÷1) don't change elements.

Sample Tasks:

  • Collect pictures/diagrams of real-life patterns (beadwork, tiles) and describe the repeating core/operation.
  • Explore patterns in different number systems (e.g., Mayan numerals).

C1.2 Create and translate patterns... using various representations...

Focus: Representing the same pattern structure in different ways (e.g., shapes, numbers, tables of values).

Examples:

  • Translate numeric pattern (1, 2, 1, 2...) into shapes (circle, square, circle, square...).
  • Translate growing shape pattern into a table of values (Term # vs. # of Tiles).

Key Concepts:

  • Same underlying structure can have different looks.
  • Table of Values: Term Number (position) vs. Term Value (quantity).

Note:

  • Comparing translations highlights the underlying math structure.

Sample Tasks:

  • Describe real-life patterns verbally ("I get older each year").
  • Translate a numeric pattern to shapes.
  • Create a growing tile pattern, translate to table of values. Answer prediction questions (How many tiles in Term 6? Which term has 16 tiles?).

C1.3 Determine pattern rules and use them to extend, predict, identify missing elements...

Focus: Generalizing patterns by finding the rule (in words) and using it to continue the pattern, find future terms, or fill in gaps.

Examples:

  • Extending: What came before? What comes next?
  • Predicting: What shape/number is in the 10th position? 100th?
  • Missing Elements: Fill in blanks in sequences or tables.

Key Concepts:

  • Patterns are repetitive.
  • Pattern Rule: A general description of how the pattern works.
  • Near Prediction: Finding the next few terms.
  • Far Prediction: Finding a term much later in the sequence (often needs calculation).

Note:

  • Generalizing (finding the rule) is key.
  • Use rules to verify predictions.

Sample Tasks:

  • Extend patterns that don't start at Term 1, forwards and backwards.
  • Analyze patterns growing in multiple directions (e.g., number pyramids).
  • Make near/far predictions for various patterns (shapes, numbers). Justify with the rule. (e.g., Circle pattern: 100th position? Can Term X have 50 sections?).
  • Fill in missing elements in sequences or tables.

C1.4 Create and describe patterns to illustrate relationships among whole numbers up to 1000.

Focus: Using patterns to show connections between operations or within the number system.

Examples:

  • Pattern showing addition/subtraction facts for 7 extended to larger numbers (e.g., $500+7=507$, $507-7=500$; $501+6=507$, $507-6=501$...).
  • Pattern showing relationship between $\times 5$ and $\div 5$ facts (e.g., $5 \times 3 = 15$, $15 \div 3 = 5$; $5 \times 4 = 20$, $20 \div 4 = 5$...).

Key Concepts:

  • Patterns reveal number relationships.
  • Many patterns exist within the number system.

Note:

  • Connects to "Number Strings" teaching strategy.

Sample Tasks:

  • Create number string patterns to show how basic facts apply to larger numbers or the inverse relationship between operations. Justify the pattern.

C2: Equations and Inequalities

▼

C2.1 Describe how variables are used, and use them...

Focus: Understanding that variables (letters or symbols) can represent unknown quantities or quantities that change in different contexts like formulas, properties, or equations.

Examples:

  • Properties: $a+b = b+a$.
  • Formulas: Perimeter of square = $s+s+s+s$ or $4s$.
  • Tables of Values: Wheels = $2 \times b$ (bicycles).
  • Unknown Quantity: $8 + \square = 16$; $n+6 = 20-5$.

Key Concepts:

  • Variables stand for numbers that can change or are unknown.

Note:

  • Identify constants vs. variables in real life.
  • Connects to Mathematical Modelling.
  • Implicitly used when finding missing addends/factors.
  • Can be letters, symbols, or words (in coding).

Sample Tasks:

  • Discuss variable use as it arises (e.g., labeling square sides 's' for perimeter).
  • Use variables for unknowns in simple equations ($n+2=7$).

C2.2 Determine whether given sets of expressions are equivalent or not.

Focus: Understanding the meaning of the equal sign (=) as "is the same as" and determining if two different calculations result in the same value.

Examples:

  • Are $3 \times 4$ and $2 \times 6$ equivalent? (Yes, both = 12).
  • Are $16 + 32$ and $50 - 2$ equivalent? (Yes, both = 48). Use models or code to verify.

Key Concepts:

  • Equivalent expressions have the same value.
  • Equal sign (=) means balance/equivalence. Not equal sign ($\neq$).
  • Use strategies/models to check equivalence.

Note:

  • Correct understanding of '=' is crucial for algebra.

Sample Tasks:

  • Compare pairs of expressions using relational rods ($3 \times 4$ vs $2 \times 6$; $4 \times 2$ vs $3 \times 3$). Create equivalent expressions if they aren't equal.
  • Compare expressions with different operations using models or simple code (like the turtle example).

C2.3 Identify and use equivalent relationships for whole numbers up to 1000...

Focus: Recognizing that a number can be represented in many equivalent ways (different expressions, decompositions) and using properties like commutative and associative to create equivalent expressions.

Examples:

  • $2 \times 6 = 6 + 6 = 2+2+2+2+2+2$.
  • Different ways to make 12 (using addition, subtraction, multiplication).
  • Commutative Property: $8 + 352 = 352 + 8$; $5 \times 2 = 2 \times 5$.
  • Associative/Commutative for Addition: $263 + 364 + 7 = 263 + 7 + 364$ (makes adding easier).

Key Concepts:

  • Equivalence: Different forms representing the same value.
  • Properties (commutative, associative) create equivalent expressions.

Note:

  • Equivalency is a fundamental math principle.

Sample Tasks:

  • Keep an "Equivalence" section in a math journal. Record examples found throughout the year.
  • Represent 12 in multiple ways using relational rods and symbolic expressions (addition, multiplication).
  • Demonstrate how commutative/associative properties make calculations easier.
